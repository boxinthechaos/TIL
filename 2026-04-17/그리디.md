# 그리디알고리즘이란?
매 순간 최적이라고 생각되는 것을 선택하여 최종적인 해답에 도달하는 알고리즘입니다

미래를 고려하지 않고 현재 상황에서 가장 저렴하거나 큰 값을 선택하는 근시안적인 방식으로, 전체 최적해를 보장하지는 않지만 빠르고 효율적으로 근사치를 구하는 데 적합합니다

## 핵심 특징 및 요약
- **탐욕적 선택**: 현재 상황에서 당장 최적인 답을 선택
- **비가역성**: 한번 선택한 것은 되돌리지 않음
- **적용 조건**: 지역적 최적해가 전체 최적해가 되는 문제, 즉 '탐욕 선택 속성'과 '최적 부분 구조'를 만족해야 함
- **대표 예시**: 거스름돈 문제(가장 큰 단위부터 계산), 최소 신장 트리(MST), 활동 선택 문제
- **장단점**: 계산 속도가 매우 빠르지만(장점), 항상 최적의 해를 보장하지는 않음(단점)

## 동작 원리
1. **선택 절차**(Selection Procedure): 현재 상태에서 최적의 해를 선택
2. **적절성 검사**(Feasibility Check): 선택된 해가 유효한지(조건 만족 여부) 검사
3. **해 선택**(Solution Check): 목표에 도달했는지 확인

### 대표적인 활용 예시
- 거스름돈 문제: 870원을 거슬러 줄 때, 가장 적은 동전 수를 구하기 위해 500원, 100원, 50원, 10원 순으로 큰 금액부터 채우는 경우
- 활동 선택 문제(Activity Selection Problem): 회의실 배정 등에서 종료 시간이 가장 빠른 활동을 먼저 선택하는 방식

## 구현 예시
```
n = int(input())  # 거슬러 줄 돈

coins = [500, 100, 50, 10]
count = 0

for coin in coins:
    count += n // coin
    n %= coin

print(count)
```
