# 면접 예상 질문 정리
## CS 관련 지식
> **웹 통신의 큰 흐름: https://www.google.com/ 에 접속할 때 생기는 일**
> 
> 브라우저가 URL에 적힌 값을 파싱해서 HTTP Request Message를 만들고 Os에 전송을 요청한다 이때 Domain으로 요청을 보낼 수 없기 때문에 DNS Lookup을 수행합니다
>
> DNS Lookup과정은 크롬의 경우라면 브라우저 -> hosts 파일 -> DNS cache의 순서로 도메인에 매칭되는 ip를 찾습니다 일반적으로 설명하는 DNS Lookup은 루트 도메인서버에서부터 서브 도메인 서버순으로 찾게 됩니다
>
> 이 요청은 프로토콜 스택이라는 OS에 저장되어 있는 네트워크 제어용 소프트웨어에 의해 패킷에 담기고 패킷에 제어정보를 덧붙여 Lan 어댑터에 전송하고 lan 어댑터는 이를 전기신호로 변화시켜 송출합니다
>
> 패킷은 스위칭 허브등을 경유하여 인터넷 접속용으로 라우터에서 ISP로 전달되어 인터넷으로 이동합니다<br>액세스 회선에 의해 통신사용 라우터로 운반되고 인터넷의 핵심부로 전달됩니다 고속 라우터들 사이로 목적지 까지 패킷이 흘러들어가게 됩니다
>
> 핵심부를 통과한 패킷은 목적이인 LAN에 도착하게 되고 방화벽이 패킷을 검사하고 캐시 서버로 보내어 웹 서버에 갈 필요가 있는지 검사합니다
>
> 웹 서버에 도착한 패킷은 프로토콜 스택이 패킷을 추출하여 메시지를 복원하고 웹 서버 애플리케이션에 넘깁니다 애플리케이션은 요청에 대한 응답 데이터를 작성하여 클라이언트로 회송하고 이는 전달된 방식 그대로 전송됩니다

> **TCP와 UDP의 차이점에 대해 설명하시오**<br>
> TCP는 연결 지향적 프로토콜이고 UDP는 데이터를 데이터그램단위로 전송하는 프로토콜입니다
>
> TCP는 가상 회선을 만들어 신뢰성을 보장하도록 하는 프로토콜로 따로 신뢰성을 보장하도록 하긴 위한 절차 때문에 절차가 없는 UDP보다 속도가 느린편입니다
>
> 그래서 TCP는 파일전송과 같은 신뢰성이 중요한 곳에 사용되고 UDP는 스트리밍, RTP와 같은 연속성이 중요한 곳에 사용이 됩니다
>
> 하지만 UDP가 신뢰성을 보장하지 않을 뿐이지 사용자가 직접 신뢰성을 보장하도록 할 수 있습니다 그래서 HTTP/3은 QUIC이라는 프로토콜 기반으로 하는데 QUIC는 UDP를 기반으로 합니다 즉 UDP 자체는 신뢰성을 보장하진 않지만 추가적인 정의를 통해 신뢰성을 보장받을 수 있습니다

> **TCP3,4 way handshake에 대해 설명하시오**<br>
> TCP 3way handshake는 가상 회선을 수립하는 단계입니다 클라이언트는 서버에 요청을 전송할 수 있는지 서버는 클라이언트에게 응답을 할 수 있는지 확인하는 과정입니다 SYN ACK 패킷을 주고받으며 임의의 난수로 SUN 플래그를 전송하고 ACK 플래그에는 1을 더한값을 전송합니다 정확한 순서로는 SYN(n) -> ACK(n+1), SYN(m) -> ACK(m+1) 순으로 일어납니다
> 왜 임의의 난수로 설정을 하냐면 기존 요청과 구분을 하기 위해서입니다
>
> TCP 4way handshake는 TCP연결을 해제하는 단계로 클라이언트는 서버에게 연결 해제를 통보하고 서버가 이를 확인하고 클라이언트에게 이를 받았음을 전송해주고 최종적으로 연결이 해제됩니다 단 서버에서 소켓이 닫혔다고 통지해도 클라이언트 측에서는 일정시간 대기를 하는데 이는 패킷이 나중에 도착 할 수 도 있기 때문이다

> **HTTP와 HTTPS의 차이점을 설명하시오**<br>
> HTTP는 따로 암호화 과정을 걸치지 않기 때문에 중간에 패킷을 가로 챌 수 있기 때문에 이를 해결하기 위해 나온 것이 HTTPS이다 HTTPS는 암호화 과정을 거쳐 패킷을 보내기 때문에 더욱 안전합니다
